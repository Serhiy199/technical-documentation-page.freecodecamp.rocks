Вступ

JavaScript — це кросплатформна об’єктно-орієнтована мова сценаріїв. Це маленька і легка мова. Усередині хост-оточення (наприклад, веб-браузера) JavaScript можна підключити до об’єктів свого середовища, щоб забезпечити програмний контроль над ними.

JavaScript містить стандартну бібліотеку об’єктів, наприклад Array, Date і Math, а також базовий набір елементів мови, таких як оператори, керуючі структури та оператори. Основний JavaScript можна розширити для різноманітних цілей, доповнивши його додатковими об’єктами; наприклад:

  • Клієнтський JavaScript розширює основну мову, надаючи об’єкти для керування браузером та його об’єктною моделлю документа (DOM). Наприклад, розширення на стороні клієнта дозволяють програмі розміщувати елементи у формі HTML і реагувати на події користувача, такі як клацання мишею, введення форми та навігація сторінкою.
  • JavaScript на стороні сервера розширює основну мову, надаючи об’єкти, які стосуються запуску JavaScript на сервері. Наприклад, розширення на стороні сервера дозволяють програмі спілкуватися з базою даних, забезпечувати безперервність інформації від одного виклику програми до іншого або виконувати маніпуляції з файлами на сервері.

JavaScript і Java

JavaScript і Java схожі в деяких аспектах, але принципово відрізняються в деяких. Мова JavaScript нагадує Java, але не має статичної типізації та жорсткої перевірки типів Java. JavaScript дотримується більшості синтаксису виразів Java, умов іменування та основних конструкцій потоку керування, що стало причиною перейменування з LiveScript на JavaScript.

На відміну від системи класів під час компіляції Java, створених за допомогою декларацій, JavaScript підтримує систему часу виконання, засновану на невеликій кількості типів даних, що представляють числові, логічні та рядкові значення. JavaScript має об’єктну модель на основі прототипу замість більш поширеної об’єктної моделі на основі класу. Модель на основі прототипу забезпечує динамічне успадкування; тобто те, що успадковується, може відрізнятися для окремих об’єктів. JavaScript також підтримує функції без будь-яких спеціальних декларативних вимог. Функції можуть бути властивостями об'єктів, що виконуються як вільно типізовані методи.

JavaScript є мовою дуже вільної форми порівняно з Java. Вам не потрібно оголошувати всі змінні, класи та методи. Вам не потрібно турбуватися про те, чи є методи публічними, приватними чи захищеними, і вам не потрібно реалізовувати інтерфейси. Змінні, параметри та типи повернення функцій не вказуються явно.

Привіт Світ

Щоб розпочати написання JavaScript, відкрийте Scratchpad і напишіть свій перший код JavaScript «Hello world»:

function greetMe(yourName) { alert("Hello " + yourName); } greetMe("World");

виберіть код у блокноті та натисніть Ctrl+R, щоб спостерігати, як він розгортається у вашому браузері!

Змінні

Ви використовуєте змінні як символічні імена для значень у своїй програмі. Імена змінних, які називаються ідентифікаторами, відповідають певним правилам.

Ідентифікатор JavaScript має починатися з літери, підкреслення (_) або знака долара ($); наступні символи також можуть бути цифрами (0-9). Оскільки JavaScript чутливий до регістру, літери включають символи від "A" до "Z" (великі літери) та символи від "a" до "z" (малі літери).

В ідентифікаторах можна використовувати літери ISO 8859-1 або Unicode, наприклад å та ü. Ви також можете використовувати керуючі послідовності Unicode як символи в ідентифікаторах. Деякі приклади офіційних назв: Number_hits, temp99 і _name.

Оголошення змінних

Ви можете оголосити змінну трьома способами:

За допомогою ключового слова var. Наприклад,

var x = 42.

цей синтаксис можна використовувати для оголошення як локальних, так і глобальних змінних.

Просто присвоївши йому значення. Наприклад,

x = 42.

This завжди оголошує глобальну змінну. Він створює суворе попередження JavaScript. Ви не повинні використовувати цей варіант.

З ключовим словом let. Наприклад,

let y = 13.

цей синтаксис можна використовувати для оголошення локальної змінної області видимості блоку. Див. Область змінної нижче.

Область змінної

Коли ви оголошуєте змінну поза будь-якою функцією, вона називається глобальною змінною, оскільки вона доступна для будь-якого іншого коду в поточному документі. Коли ви оголошуєте змінну у функції, вона називається локальною змінною, оскільки вона доступна лише в цій функції.

JavaScript до ECMAScript 2015 не має області оператора блоку; скоріше, змінна, оголошена в блоці, є локальною для функції (або глобальної області видимості), всередині якої знаходиться блок. Наприклад, наступний код буде реєструвати 5, оскільки область x — це функція (або глобальний контекст), у якій x оголошено, а не блок, який у цьому випадку є оператором if.

if (true) { var x = 5; } console.log(x); // 5

Ця поведінка змінюється під час використання оголошення let, представленого в ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined

Глобальні змінні

Глобальні змінні насправді є властивостями глобального об'єкта. На веб-сторінках глобальним об’єктом є вікно, тому ви можете встановлювати та отримувати доступ до глобальних змінних за допомогою синтаксису window.variable.

Таким чином, ви можете отримати доступ до глобальних змінних, оголошених в одному вікні або кадрі, з іншого вікна або кадру, вказавши назву вікна або кадру. Наприклад, якщо в документі оголошено змінну під назвою phoneNumber, ви можете посилатися на цю змінну з iframe як parent.phoneNumber.

Константи

Ви можете створити константу з іменем лише для читання за допомогою ключового слова const. Синтаксис постійного ідентифікатора такий самий, як і для змінного ідентифікатора: він повинен починатися з літери, підкреслення або знака долара і може містити букви, цифри або символи підкреслення.

const PI = 3.14;

Константа не може змінити значення через присвоєння або повторно оголошена під час виконання сценарію. Його потрібно ініціалізувати значенням.

Правила області видимості для констант такі ж, як і для змінних області видимості let block. Якщо ключове слово const опущено, передбачається, що ідентифікатор представляє змінну.>

Ви не можете оголосити константу з тим самим іменем, що й функція або змінна в тій самій області. Наприклад:

// THIS WILL CAUSE AN ERROR function f() {}; const f = 5; // THIS WILL CAUSE AN ERROR ALSO function f() { const g = 5; var g; //statements }

Однак атрибути об’єктів не захищені, тому наступний оператор виконується без проблем.

const MY_OBJECT = {"key": "value"}; MY_OBJECT.key = "otherValue";

Оператор if... else

Використовуйте оператор if, щоб виконати оператор, якщо логічна умова виконується. Використовуйте необов’язкову пропозицію else, щоб виконати оператор, якщо умова хибна. Інструкція if виглядає наступним чином:

if (condition) { statement_1; } else { statement_2; }

умовою може бути будь-який вираз, який має значення true або false. Перегляньте Boolean для пояснення того, що обчислюється як true і false. Якщо умова оцінюється як істинна, виконується оператор_1; інакше виконується оператор_2. оператор_1 і оператор_2 можуть бути будь-якими операторами, включаючи додаткові вкладені оператори if.

Ви також можете об’єднати оператори за допомогою else if, щоб мати кілька умов, перевірених послідовно, як показано нижче:

if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; }

У разі кількох умов буде виконано лише першу логічну умову, яка оцінюється як істина. Щоб виконати кілька інструкцій, згрупуйте їх у блочну інструкцію ({ ... }). Загалом, добре завжди використовувати блокові оператори, особливо під час вкладення операторів if:

if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; }

бажано не використовувати прості присвоєння в умовному виразі, оскільки присвоєння можна сплутати з рівністю, переглядаючи код. Наприклад, не використовуйте такий код:

if (x = y) { /* statements here */ }

Якщо вам потрібно використовувати присвоєння в умовному виразі, звичайною практикою є розміщувати додаткові дужки навколо присвоювання. Наприклад:

if ((x = y)) { /* statements here */ }

Оператор while

Оператор while виконує свої оператори, доки визначена умова виявляється істинною. Оператор while виглядає наступним чином:

while (condition) statement

якщо умова стає хибною, оператор у циклі припиняє виконання, а керування переходить до оператора, наступного за циклом.

Перевірка умови відбувається перед виконанням інструкції в циклі. Якщо умова повертає значення true, оператор виконується, і умова перевіряється знову. Якщо умова повертає false, виконання припиняється, а керування передається наступному оператору while.

Щоб виконати кілька операторів, використовуйте оператор блоку ({ ... }), щоб згрупувати ці оператори.

приклад:

Наступний цикл while повторюється, поки n менше трьох:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

З кожною ітерацією цикл збільшує n і додає це значення до x. Тому x і n приймають такі значення:

  • Після першого проходу: n = 1 і x = 1
  • Після другого проходу: n = 2 і x = 3
  • Після третього проходу: n = 3 і x = 6

Після завершення третього проходу умова n < 3 більше не виконується, тому цикл завершується.

Оголошення функцій

Визначення функції (також називається оголошенням функції або оператором функції) складається з ключового слова function, за яким слідують:

  • Ім'я функції.
  • Список аргументів функції, укладених у круглі дужки та розділених комами.
  • Інструкції JavaScript, які визначають функцію, укладені у фігурні дужки { }.

Наприклад, наступний код визначає просту функцію під назвою square:

function square(number) { return number * number; }

Функція square приймає один аргумент, який називається числом. Функція складається з одного оператора, який говорить про повернення аргументу функції (тобто числа), помноженого на самого себе. Оператор return визначає значення, яке повертає функція.

return number * number;

Примітивні параметри (такі як число) передаються функціям за значенням; значення передається функції, але якщо функція змінює значення параметра, ця зміна не відображається ні глобально, ні у функції, що викликає.
